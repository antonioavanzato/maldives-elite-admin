# Форма анкеты (maldives-elite.ru/anketa) — подключение к бэкенду

## Что сейчас (важно для безопасности)

Форма на странице `/anketa` (хостинг Wfolio, форма `id="touristForm"`) сейчас
отправляет данные **напрямую в Telegram из браузера**:

```js
fetch(`https://api.telegram.org/bot${BOT_TOKEN}/sendMessage`, { ... })
```

Из-за этого прямо в HTML-коде страницы **открыто лежат токен бота и ID канала**
(любой может их прочитать через «Просмотр кода»), а все персональные данные
(телефон, бюджет, пожелания) уходят в Telegram и **нигде не сохраняются** —
нет базы, нет админки, нет соответствия 152-ФЗ.

Новый бэкенд это исправляет: форма шлёт заявку на API Gateway → функция
`form-intake` пишет её в YDB (данные в РФ) и сама отправляет в Telegram
уведомление **только с именем**. Токен бота остаётся на сервере, в код страницы
он больше не попадает.

## Поля формы и их соответствие бэкенду

| Поле формы (`name`) | Тип | Значения | Ключ в JSON для `/submit` |
|---|---|---|---|
| `fio` | text (обяз.) | — | `client` |
| `phone` | tel (обяз.) | — | `phone` |
| `direction` | select (обяз.) | Мальдивы, Маврикий, Тайланд, Вьетнам, Дубай, Сейшелы, Бали, **other** | `destination` |
| `direction_other` | text | если `direction = other` | → в `destination` |
| `departure_date` | date (обяз.) | `ГГГГ-ММ-ДД` | `departDate` |
| `nights` | select (обяз.) | 7, 8, 10, 12, 14, **other** | `nights` |
| `nights_other` | text | если `nights = other` | → в `nights` |
| `tourists_composition` | select (обяз.) | Пара, Семья без детей, Семья с детьми, Компания друзей, Соло | `group` |
| `villa_type` | select | С бассейном, На пляже, Над водой, С садом, **other** | `villa` |
| `villa_other` | text | если `villa_type = other` | → в `villa` |
| `food` | select | Все включено, Полупансион, Завтрак, Без питания | `meal` |
| `departure_city` | select (обяз.) | Москва, СПб, Екатеринбург, Новосибирск, Казань, Краснодар, **other** | `departureCity` |
| `city_other` | text | если `departure_city = other` | → в `departureCity` |
| `reef_needed` | checkbox | вкл/выкл | `reef` (true/false) |
| `budget` | number (обяз.) | руб. | `budget` |
| `important_notes` | textarea | — | `notes` |

Триггер «другого» варианта во всех select — значение `other`.
Обязательны на бэкенде только `client` и `phone`; остальное передаётся как есть.

## Готовый обработчик отправки (заменяет отправку в Telegram)

В коде страницы `/anketa` найдите обработчик `form.addEventListener('submit', …)`,
который сейчас шлёт `fetch` на `api.telegram.org`, и замените тело отправки на
приведённое ниже. Впишите свой адрес шлюза в `API_BASE` (домен из шага 6
инструкции по деплою).

```html
<script>
(function () {
  // Адрес API Gateway из Yandex Cloud (шаг 6 инструкции по деплою):
  var API_BASE = "https://<домен-шлюза>.apigw.yandexcloud.net";

  var form = document.getElementById("touristForm");
  if (!form) return;

  // берём значение поля, а если в select выбрано "other" — из парного *_other
  function val(name, otherName) {
    var el = form.querySelector('[name="' + name + '"]');
    var v = el ? String(el.value || "").trim() : "";
    if (v === "other" && otherName) {
      var o = form.querySelector('[name="' + otherName + '"]');
      return o ? String(o.value || "").trim() : "";
    }
    return v;
  }

  form.addEventListener("submit", async function (e) {
    e.preventDefault();

    var phoneDigits = (form.querySelector('[name="phone"]').value || "").replace(/\D/g, "");
    if (phoneDigits.length < 10) {
      alert("Пожалуйста, введите корректный номер телефона (минимум 10 цифр)");
      return;
    }

    var nights = val("nights", "nights_other");
    if (/^\d+$/.test(nights)) nights = nights + " ночей"; // "7" → "7 ночей"

    var payload = {
      client:        val("fio"),
      phone:         (form.querySelector('[name="phone"]').value || "").trim(),
      destination:   val("direction", "direction_other"),
      departDate:    val("departure_date"),
      nights:        nights,
      group:         val("tourists_composition"),
      villa:         val("villa_type", "villa_other"),
      meal:          val("food"),
      departureCity: val("departure_city", "city_other"),
      reef:          !!(form.querySelector('[name="reef_needed"]') && form.querySelector('[name="reef_needed"]').checked),
      budget:        parseInt((form.querySelector('[name="budget"]').value || "0").replace(/\D/g, ""), 10) || 0,
      notes:         val("important_notes")
    };

    var btn = form.querySelector(".submit-btn") || form.querySelector('[type="submit"]');
    var original = btn ? btn.innerHTML : "";
    if (btn) { btn.innerHTML = "📤 Отправляем…"; btn.disabled = true; }

    try {
      var res = await fetch(API_BASE + "/submit", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(payload)
      });
      if (!res.ok) throw new Error("HTTP " + res.status);
      alert("Заявка отправлена! Мария свяжется с вами в ближайшее время.");
      form.reset();
    } catch (err) {
      alert("Не удалось отправить заявку. Попробуйте ещё раз или напишите в Telegram/WhatsApp.");
      console.error(err);
    } finally {
      if (btn) { btn.innerHTML = original; btn.disabled = false; }
    }
  });
})();
</script>
```

## После замены — обязательно

1. **Отключите старую отправку в Telegram** из браузера и **удалите из кода
   страницы `BOT_TOKEN` и `CHANNEL_ID`** — они больше не нужны и не должны
   лежать в публичном HTML. Токен теперь только в переменных функции `form-intake`.
2. **Смените токен бота** у @BotFather (`/revoke` → новый токен), раз старый
   уже был засвечен в открытом коде. Новый токен пропишите только в переменной
   `TG_BOT_TOKEN` функции `form-intake`.
3. Проверьте отправку тестовой заявки (шаг 9 инструкции по деплою): заявка
   должна появиться в YDB и прийти уведомлением в канал.
