# 🚀 Простая инструкция: что создать в Yandex Cloud и какой файл взять

Делай сверху вниз. Слева — что создаёшь в консоли https://console.yandex.cloud,
справа — какой файл из репозитория (папки `deploy` и `backend`) для этого нужен.

Если застрял на шаге — открой подробный файл `ИНСТРУКЦИЯ-ДЕПЛОЙ.md` (там тот же
порядок, но с деталями).

---

## Шаг 1. Сервисный аккаунт
- Консоль → **Сервисные аккаунты** → **Создать**. Имя: `maldives-sa`.
- Дай ему роли: `ydb.editor` и `functions.functionInvoker`.
- 📋 Скопируй его **ID** (строка `aje...`) — запиши в блокнот.
- 📄 Файл не нужен.

## Шаг 2. База данных
- Консоль → **YDB** → **Создать базу** → тип **Serverless**. Имя: `maldives-db`.
- Открой базу → запиши **Эндпоинт** и **Расположение базы** (путь `/ru-central1/...`).
- Открой в базе **SQL-запрос**, вставь весь текст файла и нажми «Выполнить».
- 📄 Берёшь файл: **`backend/schema.sql`**

## Шаг 3. Telegram (для уведомлений)
- В Telegram: @BotFather → `/newbot` → запиши **токен бота**.
- Создай закрытый канал, добавь бота админом, узнай **ID канала**.
- 📄 Как узнать ID канала — в файле: **`ПЕРЕМЕННЫЕ-И-СЕКРЕТЫ.md`**

## Шаг 4. Секреты для входа в админку
- Открой Терминал на Mac, выполни 2 команды из файла (пароль + секрет сессий).
- 📄 Команды в файле: **`ПЕРЕМЕННЫЕ-И-СЕКРЕТЫ.md`** (раздел «Команды генерации»)

## Шаг 5. (по желанию) Ключи для push-уведомлений
- В Терминале выполни: `npx web-push generate-vapid-keys` → запиши 2 ключа.
- 📄 Подробно в файле: **`PUSH-УВЕДОМЛЕНИЯ.md`**
- Не нужны push сейчас? Пропусти этот шаг.

## Шаг 6. Собрать архивы функций
- В Терминале: `cd backend && bash build-functions.sh`
- Появятся 2 файла: `backend/dist/form-intake.zip` и `backend/dist/admin-api.zip`.
- 📄 Скрипт: **`backend/build-functions.sh`** (лежит в этой же папке)

## Шаг 7. Функция №1 — приём заявок
- Консоль → **Cloud Functions** → **Создать функцию**. Имя: `form-intake`.
- Среда: **Node.js 18**. Загрузи ZIP. Точка входа: `index.handler`.
- Сервисный аккаунт: `maldives-sa`.
- Впиши переменные: `YDB_ENDPOINT`, `YDB_DATABASE`, `TG_BOT_TOKEN`, `TG_CHAT_ID`
  (+ 3 VAPID-переменные, если делаешь push).
- 📋 Запиши **ID функции**.
- 📄 Берёшь архив: **`backend/dist/form-intake.zip`**. Список переменных: **`ПЕРЕМЕННЫЕ-И-СЕКРЕТЫ.md`**

## Шаг 8. Функция №2 — админка (API)
- То же самое. Имя: `admin-api`. ZIP другой. Точка входа: `index.handler`.
- Сервисный аккаунт: `maldives-sa`.
- Впиши переменные: `YDB_ENDPOINT`, `YDB_DATABASE`, `ADMIN_EMAIL`,
  `ADMIN_PASSWORD_HASH`, `SESSION_SECRET`.
- 📋 Запиши **ID функции**.
- 📄 Берёшь архив: **`backend/dist/admin-api.zip`**

## Шаг 9. API Gateway (шлюз)
- Сначала открой файл в редакторе и замени в нём 3 вещи: ID функции form-intake,
  ID функции admin-api, ID сервисного аккаунта (везде, где `<...>`).
- Консоль → **API Gateway** → **Создать** → вставь весь текст файла.
- 📋 Открой шлюз → скопируй **Служебный домен** (`https://...apigw.yandexcloud.net`).
  Это адрес API — он нужен в шагах 10 и 11.
- 📄 Берёшь файл: **`backend/api-gateway.yaml`**

## Шаг 10. Разместить админку (сайт-панель)
- Скачай из GitHub-репозитория админки файлы: `index.html`, `manifest.json`,
  `sw.js`, папку `images`.
- В `index.html` впиши адрес шлюза (из шага 9) в строку `const API_BASE = "..."`.
  (Делаешь push? Ещё впиши публичный VAPID-ключ в строку `const VAPID_PUBLIC_KEY` — код подписки уже встроен, см. `PUSH-УВЕДОМЛЕНИЯ.md`.)
- Консоль → **Object Storage** → **Создать бакет** (публичный) → загрузи файлы →
  включи «Хостинг сайта», главная `index.html`.
- 📋 Скопируй адрес бакета — это ссылка на админку для Марии.
- 📄 Инструкция подробно: **`ИНСТРУКЦИЯ-ДЕПЛОЙ.md`** (шаг 7)

## Шаг 11. Подключить форму сайта
- На странице `/anketa` замени отправку формы на готовый код, впиши в него
  адрес шлюза (из шага 9).
- 📄 Готовый код и что удалить: **`ФОРМА-АНКЕТА.md`**

## Шаг 12. Проверка
- Отправь тестовую заявку через форму → должна прийти в Telegram и появиться
  в админке. Push (если делал) — прилетит на телефон.
- 📄 Что проверять: **`ИНСТРУКЦИЯ-ДЕПЛОЙ.md`** (шаг 9)

---

## Шпаргалка «что создаём в консоли»
1. Сервисный аккаунт `maldives-sa`
2. База YDB `maldives-db`
3. Функция `form-intake`
4. Функция `admin-api`
5. API Gateway `maldives-gw`
6. Бакет Object Storage (для админки)

## Шпаргалка «какой файл для чего»
| Файл | Для чего |
|---|---|
| `backend/schema.sql` | вставить в SQL-запрос базы (шаг 2) |
| `backend/build-functions.sh` | собрать zip-архивы функций (шаг 6) |
| `backend/dist/form-intake.zip` | загрузить в функцию form-intake (шаг 7) |
| `backend/dist/admin-api.zip` | загрузить в функцию admin-api (шаг 8) |
| `backend/api-gateway.yaml` | вставить в API Gateway (шаг 9) |
| `ПЕРЕМЕННЫЕ-И-СЕКРЕТЫ.md` | откуда брать значения переменных |
| `ФОРМА-АНКЕТА.md` | подключить форму сайта (шаг 11) |
| `PUSH-УВЕДОМЛЕНИЯ.md` | если нужны push (шаги 5, 10) |
| `ИНСТРУКЦИЯ-ДЕПЛОЙ.md` | подробная версия всех шагов |
