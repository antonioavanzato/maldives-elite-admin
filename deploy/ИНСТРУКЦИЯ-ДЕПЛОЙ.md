# Развёртывание админки Maldives Elite на Yandex Cloud — пошагово

Полная инструкция «с нуля». Рассчитана на выполнение через **веб-консоль**
Яндекс Облака (https://console.yandex.cloud) — CLI не обязателен.
Делайте строго по порядку, шаги зависят друг от друга.

## Что в итоге получится (архитектура)

```
Форма на сайте ──POST /submit──▶ API Gateway ──▶ функция form-intake ──▶ YDB (все данные, РФ)
                                                        └──▶ Telegram-канал (только имя)

Админка (PWA) ──/login, /leads──▶ API Gateway ──▶ функция admin-api ──▶ YDB
```

- **YDB (Serverless)** — база заявок, данные хранятся в дата-центрах РФ (152-ФЗ).
- **2 облачные функции** (Node.js) — приём заявок и API админки.
- **API Gateway** — единая точка входа, маршрутизирует запросы к функциям.
- **Object Storage** — статичный хостинг самой админки (PWA).

## Состав папки

| Файл | Назначение |
|---|---|
| `backend/schema.sql` | таблица `leads` — выполнить в YDB |
| `backend/form-intake/` | код функции приёма заявок |
| `backend/admin-api/` | код функции API админки |
| `backend/api-gateway.yaml` | спецификация API Gateway (правится перед загрузкой) |
| `ПЕРЕМЕННЫЕ-И-СЕКРЕТЫ.md` | все env-переменные и как сгенерировать секреты |
| `ФОРМА-АНКЕТА.md` | разбор формы `/anketa` + готовый обработчик отправки |
| `PUSH-УВЕДОМЛЕНИЯ.md` | подключение push в PWA-админке (VAPID, подписка) |
| `backend/build-functions.sh` | сборка ZIP-архивов функций |
| `backend-README.md` | исходная заметка по архитектуре из репозитория |

> Сама админка-фронтенд (index.html, manifest.json, sw.js, images) лежит в
> GitHub-репозитории `antonioavanzato/maldives-elite-admin` в корне.
> Её деплой — шаг 7.

# ШАГ 0. Подготовка (5 минут)

1. Войдите в консоль: https://console.yandex.cloud — под аккаунтом заказчика
   (или своим, если заказчик добавил вас в облако).
2. Убедитесь, что к облаку **привязан платёжный аккаунт** (иначе функции и YDB
   не запустятся). Биллинг → должен быть статус «Активен» или «Пробный период».
3. Запомните имя **каталога** (folder), в котором будете работать
   (по умолчанию `default`). Все ресурсы создавайте в одном каталоге.

---

# ШАГ 1. Сервисный аккаунт (2 минуты)

Функциям нужен сервисный аккаунт, от имени которого они пишут в YDB.

1. Консоль → каталог → слева **«Сервисные аккаунты»** → **«Создать»**.
2. Имя: `maldives-sa`. Создать.
3. Открыть его → **«Назначить роли на каталог»** → добавить роли:
   - `ydb.editor` — чтение/запись в базу;
   - `functions.functionInvoker` — чтобы API Gateway мог вызывать функции;
   - `serverless.functions.invoker` (если предыдущей роли нет в списке —
     ищите похожую с «invoker»).
4. Скопируйте **ID сервисного аккаунта** (строка `aje…`) — пригодится в шаге 6.

---

# ШАГ 2. База YDB и таблица (10 минут)

1. Консоль → каталог → слева **«Managed Service for YDB»** (или «YDB») → **«Создать базу данных»**.
2. Тип — **Serverless**. Имя: `maldives-db`. Создать. Дождитесь статуса «Running».
3. Откройте базу. На вкладке **«Обзор»** скопируйте и сохраните:
   - **Эндпоинт** → это `YDB_ENDPOINT`
     (обычно `grpcs://ydb.serverless.yandexcloud.net:2135`);
   - **Расположение базы данных** (путь вида `/ru-central1/b1g…/etn…`)
     → это `YDB_DATABASE`.
4. В базе откройте вкладку **«Навигация»** → кнопка **«SQL-запрос»** (или «Query»).
5. Откройте файл `backend/schema.sql` из этой папки, скопируйте его содержимое целиком
   в окно запроса и нажмите **«Выполнить»**. Должна появиться таблица `leads`.

---

# ШАГ 3. Telegram-бот и канал (5 минут)

Нужно только для уведомлений о новых заявках.

1. В Telegram напишите @BotFather → `/newbot` → задайте имя → получите **токен бота**
   (`123456:AA…`) → это `TG_BOT_TOKEN`.
2. Создайте закрытый канал (или используйте существующий канал заявок).
3. Добавьте бота в канал **администратором**.
4. Опубликуйте в канале любое сообщение.
5. Откройте в браузере: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
6. Найдите `"chat":{"id":-100XXXXXXXXXX}` → число с минусом это `TG_CHAT_ID`.

(Подробнее — в `ПЕРЕМЕННЫЕ-И-СЕКРЕТЫ.md`.)

# ШАГ 4. Сгенерировать секреты админки (2 минуты)

В Терминале на Mac (замените пароль на настоящий):

```bash
printf '%s' 'ПарольМарии' | shasum -a 256 | awk '{print $1}'   # → ADMIN_PASSWORD_HASH
openssl rand -hex 32                                            # → SESSION_SECRET
```

Сохраните обе строки. Пароль передайте Марии отдельно (в коде он не хранится).

# ШАГ 5. Загрузить функции (15 минут)

## 5.1. Собрать архивы

В Терминале:
```bash
cd backend
bash build-functions.sh
```
Появятся `backend/dist/form-intake.zip` и `backend/dist/admin-api.zip`.

> Если `npm` не установлен — поставьте Node.js LTS с https://nodejs.org,
> затем повторите. Либо соберите архивы вручную: в каждой папке функции
> выполните `npm install --omit=dev`, затем заархивируйте `index.js`,
> `package.json` и `node_modules` в один zip.

## 5.2. Функция `form-intake`

1. Консоль → **«Cloud Functions»** → **«Создать функцию»**. Имя: `form-intake`. Создать.
2. **«Редактировать»** → Среда выполнения: **Node.js 18** (или новее).
3. Способ: **«ZIP-архив»** → загрузите `backend/dist/form-intake.zip`.
4. **Точка входа**: `index.handler`
5. Таймаут: `30` сек. Память: `256 МБ`.
6. **Сервисный аккаунт**: выберите `maldives-sa` (из шага 1).
7. **Переменные окружения** — добавьте:
   - `YDB_ENDPOINT`, `YDB_DATABASE` (из шага 2)
   - `TG_BOT_TOKEN`, `TG_CHAT_ID` (из шага 3)
   - `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT` — **если нужны
     push-уведомления** (см. `PUSH-УВЕДОМЛЕНИЯ.md`). Без них push просто
     не шлётся, всё остальное работает.
8. **«Создать версию»**. Скопируйте **ID функции** (`d4e…`) — понадобится в шаге 6.

## 5.3. Функция `admin-api`

Повторите то же самое:
1. Создать функцию `admin-api`, Node.js 18, ZIP `backend/dist/admin-api.zip`,
   точка входа `index.handler`, сервисный аккаунт `maldives-sa`.
2. Переменные окружения:
   - `YDB_ENDPOINT`, `YDB_DATABASE` (из шага 2)
   - `ADMIN_EMAIL` (почта Марии)
   - `ADMIN_PASSWORD_HASH`, `SESSION_SECRET` (из шага 4)
3. Создать версию. Скопируйте **ID функции** `admin-api`.

# ШАГ 6. API Gateway (10 минут)

1. Откройте файл `backend/api-gateway.yaml` в этой папке в текстовом редакторе.
2. Замените плейсхолдеры на реальные значения:
   - `<FUNCTION_ID_FORM_INTAKE>` → ID функции `form-intake` (шаг 5.2);
   - `<FUNCTION_ID_ADMIN_API>` → ID функции `admin-api` (шаг 5.3);
   - `<SERVICE_ACCOUNT_ID>` → ID сервисного аккаунта `maldives-sa` (шаг 1).
   Плейсхолдеров несколько — замените **каждое** вхождение (в редакторе:
   «Заменить всё»).
3. Консоль → **«API Gateway»** → **«Создать API-шлюз»**. Имя: `maldives-gw`.
4. В поле спецификации **удалите шаблон** и вставьте полностью
   отредактированный `backend/api-gateway.yaml`. Создать.
5. Откройте шлюз → скопируйте **«Служебный домен»**
   (вид `https://d5d….apigw.yandexcloud.net`). Это ваш `API_BASE`.

Проверка (в Терминале, подставьте домен):
```bash
curl -i "https://<домен-шлюза>/leads"
```
Ожидается `401 unauthorized` — это нормально (нет токена). Значит маршрут жив.

# ШАГ 7. Разместить админку (Object Storage) (15 минут)

Админка — это статичный PWA из корня репозитория
`antonioavanzato/maldives-elite-admin` (файлы `index.html`, `manifest.json`,
`sw.js`, папка `images/`).

## 7.1. Прописать адрес API в админке

1. Скачайте `index.html` админки из репозитория.
2. Найдите в нём константу `API_BASE` (в начале скрипта) и впишите домен
   шлюза из шага 6.5, например:
   ```js
   const API_BASE = "https://d5d….apigw.yandexcloud.net";
   ```
3. Сохраните.

## 7.2. Создать бакет и загрузить файлы

1. Консоль → **«Object Storage»** → **«Создать бакет»**.
   Имя: `maldives-admin` (глобально уникальное; при занятости добавьте суффикс).
2. Доступ: **«Публичный»** для чтения объектов.
3. Загрузите в бакет: `index.html`, `manifest.json`, `sw.js` и папку `images/`.
4. Вкладка **«Веб-сайт»** (хостинг сайта): включите, главная страница `index.html`.
5. Скопируйте адрес сайта бакета
   (вид `https://maldives-admin.website.yandexcloud.net`) — это ссылка на админку.

## 7.3. Сузить CORS (безопасность)

1. В `backend/admin-api/index.js` замените
   `"Access-Control-Allow-Origin": "*"` на адрес админки из 7.2.
2. Пересоберите архив (`bash build-functions.sh`) и загрузите новую версию
   функции `admin-api` (Cloud Functions → admin-api → Редактировать → новый ZIP
   → Создать версию).

# ШАГ 8. Подключить форму анкеты (10 минут)

Форма заявки живёт на странице `maldives-elite.ru/anketa` (Wfolio, форма
`id="touristForm"`). **Сейчас она шлёт данные напрямую в Telegram из браузера,
и токен бота открыто лежит в коде страницы** — это надо заменить.

Полный разбор полей формы, таблица соответствия «поле формы → ключ JSON» и
**готовый обработчик отправки** (копипаст) — в файле **`ФОРМА-АНКЕТА.md`**.

Кратко:
1. В коде страницы `/anketa` найдите обработчик `form.addEventListener('submit', …)`,
   который делает `fetch` на `api.telegram.org`, и замените его на готовый
   скрипт из `ФОРМА-АНКЕТА.md` (там же вписывается `API_BASE` = домен шлюза из шага 6).
2. **Удалите из кода страницы `BOT_TOKEN` и `CHANNEL_ID`** — теперь они не нужны.
3. **Смените токен бота** у @BotFather (`/revoke`), раз старый был засвечен в
   открытом коде, и пропишите новый только в переменной `TG_BOT_TOKEN`
   функции `form-intake`.

Имена полей формы (`fio`, `direction`, `tourists_composition`…) **не совпадают**
с ключами бэкенда (`client`, `destination`, `group`…) — готовый обработчик уже
делает правильный маппинг, вручную сопоставлять не нужно.

# ШАГ 9. Финальная проверка (5 минут)

1. **Форма → база:** отправьте тестовую заявку через форму сайта. В Telegram-канал
   должно прийти «🌴 Новая заявка: Имя». В YDB (Навигация → таблица `leads`) —
   появиться строка.
2. **Админка:** откройте адрес админки (шаг 7.2), войдите под `ADMIN_EMAIL`
   и паролем Марии. Должен открыться список заявок с тестовой строкой.
3. **Действия:** смените статус заявки, удалите тестовую. Изменения должны
   отражаться в YDB.
4. **Push (если подключали):** в админке нажмите колокольчик → разрешите
   уведомления → отправьте ещё одну тестовую заявку → на устройство должен
   прийти push «Новая заявка: Имя». Детали — в `PUSH-УВЕДОМЛЕНИЯ.md`.

Если пункты прошли — система развёрнута.

# Частые проблемы

| Симптом | Причина / решение |
|---|---|
| Функция отвечает `500`, в логах «YDB driver not ready» | Неверные `YDB_ENDPOINT`/`YDB_DATABASE` или у сервисного аккаунта нет роли `ydb.editor`. |
| Админка не логинится (`401`) | Хэш пароля сгенерирован с лишним переносом строки. Используйте `printf '%s'` (без `echo`), как в шаге 4. |
| Заявка не доходит в Telegram, но в базе есть | Неверный `TG_BOT_TOKEN`/`TG_CHAT_ID` или бот не админ канала. Приём заявки при этом не ломается — уведомление опционально. |
| Браузер блокирует запрос из админки (CORS) | В `admin-api` `Access-Control-Allow-Origin` не совпадает с адресом админки. Поправьте и передеплойте (шаг 7.3). |
| `curl /leads` даёт `404`, а не `401` | В `backend/api-gateway.yaml` не заменены ID функций или неверный маршрут. Проверьте спецификацию шлюза (шаг 6). |

# Порядок «на одном экране» (шпаргалка)

1. Сервисный аккаунт `maldives-sa` + роли `ydb.editor`, `functions.functionInvoker`.
2. YDB Serverless → выполнить `backend/schema.sql` → сохранить endpoint и database.
3. Telegram: бот (token) + канал (chat_id).
4. Секреты: sha256 пароля + `openssl rand -hex 32`.
5. `bash build-functions.sh` → загрузить 2 функции с env-переменными → сохранить их ID.
6. Отредактировать `backend/api-gateway.yaml` (вставить ID) → создать шлюз → сохранить домен = API_BASE.
7. Прописать API_BASE в админке → залить в Object Storage (публичный, веб-хостинг).
8. Сузить CORS в admin-api → передеплой.
9. Форму сайта направить на `POST {домен-шлюза}/submit`.
10. Проверка: заявка → Telegram+YDB → вход в админку → смена статуса.
