# Переменные окружения и секреты

Эти значения задаются **в консоли Яндекса** в настройках каждой функции
(раздел «Переменные окружения»), а **не в коде**. Ниже — что куда и как получить.

---

## Функция `form-intake` (приём заявок)

| Переменная | Значение | Откуда взять |
|---|---|---|
| `YDB_ENDPOINT` | `grpcs://ydb.serverless.yandexcloud.net:2135` | Страница базы YDB → «Эндпоинт» (обычно именно такой) |
| `YDB_DATABASE` | `/ru-central1/b1g…/etn…` | Страница базы YDB → «Расположение базы данных» |
| `TG_BOT_TOKEN` | токен Telegram-бота | От @BotFather при создании бота |
| `TG_CHAT_ID` | ID закрытого канала уведомлений | См. ниже «Как узнать TG_CHAT_ID» |

## Функция `admin-api` (вход + управление заявками)

| Переменная | Значение | Откуда взять |
|---|---|---|
| `YDB_ENDPOINT` | тот же, что выше | — |
| `YDB_DATABASE` | тот же, что выше | — |
| `ADMIN_EMAIL` | почта Марии для входа | выбираете сами, напр. `maria@maldives-elite.ru` |
| `ADMIN_PASSWORD_HASH` | sha256-хэш пароля | команда ниже |
| `SESSION_SECRET` | случайная строка (подпись сессий) | команда ниже |

---

## Команды генерации секретов (выполнить в Терминале на Mac)

**Хэш пароля** (замените `ПарольМарии` на настоящий пароль):
```bash
printf '%s' 'ПарольМарии' | shasum -a 256 | awk '{print $1}'
```
Полученную длинную строку вставьте в `ADMIN_PASSWORD_HASH`.
Сам пароль нигде не сохраняется — передайте его Марии отдельно (в мессенджере/лично).

**Секрет сессий:**
```bash
openssl rand -hex 32
```
Строку вставьте в `SESSION_SECRET`.

## Как узнать TG_CHAT_ID закрытого канала

1. Создайте бота у @BotFather → получите `TG_BOT_TOKEN`.
2. Добавьте бота в закрытый канал **администратором**.
3. Опубликуйте в канале любое сообщение.
4. Откройте в браузере (подставив токен):
   `https://api.telegram.org/bot<TG_BOT_TOKEN>/getUpdates`
5. В ответе найдите `"chat":{"id":-100XXXXXXXXXX,...}` — это число (с минусом)
   и есть `TG_CHAT_ID`.

## Важно про безопасность

- После деплоя сузьте CORS: в `backend/admin-api/index.js` в объекте `CORS`
  замените `"Access-Control-Allow-Origin": "*"` на реальный адрес админки
  (например URL из Object Storage или домен админки) и передеплойте функцию.
- В `form-intake` CORS уже жёстко привязан к `https://maldives-elite.ru` —
  если форма живёт на другом домене, поправьте там же.
