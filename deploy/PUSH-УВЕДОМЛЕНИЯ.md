# Push-уведомления в PWA-админке

Когда с формы приходит заявка, Марии падает push «Новая заявка: Имя» — даже если
админка закрыта (установлена как PWA, уведомления разрешены).

```
Новая заявка ─▶ form-intake ─┬─▶ YDB (заявка)
                             ├─▶ Telegram (имя)
                             └─▶ Web Push всем устройствам из push_subs ─▶ телефон Марии
```

Push шлётся по протоколу Web Push с VAPID-ключами. Приватный ключ лежит только
в переменной функции. В push уходит **только имя**, без перс. данных.

## Код уже в репозитории — трогать не нужно

- `sw.js` — обработчики `push` и `notificationclick` (были изначально).
- `backend/schema.sql` — таблица `push_subs`.
- `backend/admin-api/index.js` — эндпоинт `POST /push/subscribe`.
- `backend/form-intake/index.js` — отправка push после заявки (+ зависимость `web-push`).
- `backend/api-gateway.yaml` — маршрут `/push/subscribe`.
- `index.html` — подписка на push **уже встроена** (кнопка-колокольчик +
  авто-переподписка после входа).

## Что нужно сделать при деплое — всего два действия

### 1. Сгенерировать VAPID-ключи
```bash
npx web-push generate-vapid-keys
```
Получишь `Public Key` и `Private Key`.

### 2. Вписать ключи в двух местах
- **В функцию `form-intake`** (переменные окружения): `VAPID_PUBLIC_KEY`,
  `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT` (`mailto:твоя-почта`).
- **В `index.html` админки** — строку `const VAPID_PUBLIC_KEY = "";` заполни
  тем же **публичным** ключом. Приватный во фронт не вставляй.

Всё. Больше во фронте ничего править не надо — подписка сработает сама.

## Проверка

1. Открой админку как **установленное PWA** (на iPhone — только из иконки на
   экране «Домой», не из вкладки Safari), войди.
2. Нажми колокольчик → разреши уведомления → «Push включён ✓».
3. Отправь тестовую заявку через форму `/anketa`.
4. На устройство придёт push «Новая заявка: Имя», тап открывает админку.

## Ограничения (чтобы не удивляться)

- **iPhone/iPad**: web push работает только для PWA на экране «Домой», iOS 16.4+.
  В обычной вкладке Safari push не приходит.
- **Android/десктоп** (Chrome, Edge, Firefox): работает и во вкладке, и в PWA.
- **Нужен HTTPS**: service worker и push работают только по https. Разместить
  админку нужно на HTTPS-адресе (см. `ИНСТРУКЦИЯ-ДЕПЛОЙ.md`, размещение админки).
- Push и Telegram работают параллельно — если push не дошёл (например, iPhone
  без установленного PWA), Telegram-уведомление придёт в любом случае.
